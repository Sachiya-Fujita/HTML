学んだことをメモメモ...
  • shared_ptr:参照カウント方式(0になったら勝手に解放)
    → ポインタがスコープを外れた時に自動的にdeleteしたい
    → RefPtr<Ref*>を用いて自力で参照カウントを操作する。
    → C++11からshared_ptr導入されてみんなはぴはぴっ！
  • NodeとSpriteの違いは何なのか？
    → Nodeは何も描画できないがaddchild()できて子は描画される
    → SpriteとかはNodeを継承している
  • Node→AnchorPostionデフォルトで左下
    Sprite→AnchorPostionデフォルトで中心
  • classとstructの違いはデフォルトスコープ
    classはprivate structはpublic
  • 継承の際デフォルトスコープを変えられる
    → class X:public Y はYのスコープと変わらず
    → class X:private Y はYのpublicはprivateに変わる
  • 左辺値参照 hoge&,右辺値参照 hoge&&→move, auto
    →&&→ テンプレート変数やautoに&&をつけた場合には、通常の右辺値参照とは異なる動作をする

後輩力で成長するには

後輩力とは

後輩力とはなんなのだろうか？正直私にもわからない。
調べてみたらいくつかの記事となんと書籍まで...ふむふむ
一応リンクを貼っておこう。いつか読まなきゃな〜。
  • ゼロから最強の人脈をつかむ 後輩力
  • 後輩力 凡人の僕が、友だち5000人になれた秘けつ
サイトでの表現を引用すると、”相手の先手で動き、無邪気で、謙虚な姿勢を忘れず、先輩と一緒に朝まで飲みに行けて、締めのラーメンまで食べにいけることです（帰宅後のお礼メールも忘れずに）。喜んでオゴってあげたい！と思わせてくれる人は、後輩力の高い人。” 引用先のリンク
なるほど、まぁコミニケーション能力の1つかな？気遣いができて、謙虚で素直な人材か...とても聞こえがいい。
次節以降で後輩力にこだわる理由を述べていくので良かったら→

なぜ後輩力なのか？

なぜ後輩力、後輩力と連呼しているのか。それは自分にはまだそれしかないからだ。
いや、後輩力が本当にあるのかはわからない。
だが、尊敬できるCAの人事の方に「圧倒的後輩力で成長してきてくれ！」と言われて、なんかしっくりきたのは覚えている。
私は、新しい人に会うのが好きだし、飲み会やイベントを企画することも好きだ。
気遣いはあまりしている自覚はないけど、初対面の人や目上の方々と話すのは得意な気がしている。
「コミュ力高いね！」と言われることが多い。(実際頑張ってキャラを演じている時もあるよ当然)
まぁ目上の方には気に入られることが多い...かもしれない(同期とかには怖がられることもあったけどそれはまたいずれ)
とにかく私は後輩力で成長していきたい、いくしかない。どうしていこうか。。。

後輩力で何ができるか

今まで何度も述べてきたことだが私は技術力がない。経験が少ない。当然だ、プログラミングを始めたのは2018年の12月なのだから。
おそらく今の所CAのエンジニアの中でもっとも技術と経験がない人材だろう。
だからといってこれを言い訳にしたいとは思っていないし、いつまでもこのままでいる気はない。
これらの欠点を自分の持ってる数少ない力、つまり後輩力で補っていきたいと考えている。
先輩方や技術の高い同期との交流を誰よりもとって、経験をするチャンスを掴む。
たくさん話を聞いてモチベを上げる、inputをしていく。
常に謙虚な姿勢て人の話をよく聞き、質問をする。素直に学ぶ。
幸いにもCAの社風に後輩力、素直さはあっているのだと思う。
だからこそ運よく？幸いにも？内定を頂けた。(心から感謝を述べたい。)
このチャンスを生かし、広げていくためにも後輩力を元に挑戦を続けていきたい。
今回はこんなところで締めたいと思う。
